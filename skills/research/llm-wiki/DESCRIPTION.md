Build and maintain a persistent personal knowledge base using Karpathy's LLM Wiki pattern — three-layer architecture (raw sources, wiki, schema) with incremental compounding. The wiki is a git-tracked directory of markdown files that grows smarter with every source ingested and every question answered. Unlike RAG (retrieve-at-query-time), the wiki is a compounding artifact — information is synthesized, cross-referenced, and filed once, not re-extracted every time. No external dependencies — uses only file tools, web extract, and shell commands.
